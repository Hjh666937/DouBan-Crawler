# 豆瓣电影爬虫 — 使用说明

## 一、项目简介

本项目是基于 **Scrapy** 的豆瓣电影数据采集系统，支持：

- 采集电影基本信息、评分、热门短评
- 导出 CSV 文件（`output/movies.csv`、`output/reviews.csv`）
- 导入 **SQL Server** 数据库
- 通过 **Web 控制台** 可视化操作

---

## 二、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| Python | 3.8 及以上 |
| SQL Server | 可选，用于数据库存储 |
| 网络 | 可访问豆瓣网站 |

## 四、启动 Web 控制台（推荐）

**方式一：双击运行**

双击项目根目录下的 **`启动Web控制台.bat`**

**方式二：终端运行**

```bash
python web/server.py
```

启动成功后，终端会提示：

```
请在浏览器中打开以下地址：

     >>>  http://127.0.0.1:5001  <<<
```

程序会自动尝试打开浏览器；若未自动打开，请手动复制上述地址到 Chrome / Edge 地址栏访问。

### 界面功能

| 功能区 | 说明 |
|--------|------|
| **数据概览** | 已爬电影数、短评数、类型数量 |
| **电影浏览** | 海报卡片网格、按类型筛选、搜索片名/导演/演员 |
| **电影详情** | 点击卡片查看简介、评分、热门短评 |

| **数据库** | 在「系统设置」中配置连接 |
| **数据库导入** | 创建库表、导入 CSV |
| **运行日志** | 左侧实时显示爬虫输出 |

## 五、Cookie 配置（重要）

1. 浏览器登录 [豆瓣电影](https://movie.douban.com)
2. 按 **F12** → **Network（网络）** → 刷新页面
3. 点击任意请求 → **Request Headers** → 复制 **Cookie**
4. 在 Web 界面「爬取任务」→ **豆瓣 Cookie** 中粘贴并保存  

或手动写入文件：`douban_crawler/cookie.txt`（仅保留 Cookie 一行）

### 爬取中更换 Cookie

1. 点击 **停止爬取**（或 **停止并保存 Cookie**，会同时停止并写入新 Cookie）
2. 从浏览器复制新的 Cookie 粘贴到输入框
3. 点击 **保存 Cookie**（若未用「停止并保存」）
4. 再次点击 **开始爬取** 继续采集  

> 停止后已爬数据会保留在 `output/movies.csv` 和 `output/reviews.csv` 中。

> Cookie 相当于登录凭证，请勿泄露或上传到 GitHub。

## 七、输出文件

| 文件 | 说明 |
|------|------|
| `output/movies.csv` | 电影信息，每部一行（多次爬取增量合并） |
| `output/reviews.csv` | 短评，每部最多 10 条（多次爬取增量合并） |

**movies.csv 字段：** movie_id, name, poster, summary, director, screenwriter, actors, genre, country, language, release_date, runtime, imdb_link, aka, rating, rating_count, short_review_count

**reviews.csv 字段：** movie_id, movie_name, reviewer, review_time, content, votes

## 八、数据库配置

编辑 `database/db_config.json`：

```json
{
  "server": "localhost",
  "port": 1433,
  "database": "DoubanMovies",
  "user": "sa",
  "password": "你的密码",
  "use_windows_auth": false
}
```

也可在 Web 界面「数据库连接配置」中填写并保存。

## 九、采集建议

1. **先小规模测试**：10 或 100 部，确认 CSV 正常  
2. **分批采集**：500~1000 部/批，批间休息 1~2 小时；**新爬取不会清空旧 CSV**，按 `movie_id` 自动合并  
3. **目标数量含义**：填写的数量为 CSV 中电影**总数**（若已有 200 部、填 500，则本次再采 300 部）  
4. **配置 Cookie**：长时间采集务必配置  
5. **避免休眠**：爬取期间电脑勿休眠、断网  
6. **403 过多时**：点击「停止爬取」→ 等待或更新 Cookie → 重新开始  

### 耗时参考

| 数量 | 预计耗时 |
|------|----------|
| 10 部 | 约 1 分钟 |
| 500 部 | 约 50 分钟 |
| 10000 部 | 约 15~20 小时 |

## 十一、常见问题

**Q：出现大量 403？**  
A：配置 Cookie，加大间隔，暂停 1~2 小时后分批重试。

**Q：movies.csv 为空？**  
A：爬虫异常中断时可能未写入；需等爬虫正常结束，或检查是否大量 403。

**Q：数据库连接失败？**  
A：确认 SQL Server 已启动，`db_config.json` 账号密码正确。

**Q：IMDb 字段为空？**  
A：豆瓣 API 不提供该字段，属正常情况。

**Q：Web 页面打不开？**  
A：确认已执行 `python web/server.py`，访问 http://127.0.0.1:5001
